Хеш: e6774c2
Дата: 2026-02-05
Автор: Гриша Шкробов
Описание: fix: рефактор esp_system, улучшение логгирования

Общий обзор
- Выполнена комплексная рефакторизация микросервиса esp_service с целью улучшения надежности, устранения дублирования кода, добавления валидации данных и стандартизации логирования.

1) MQTT Layer (mqtt.py)
- Добавлена валидация входящих payload'ов
- Входящие MQTT сообщения теперь валидируются через Pydantic модели (TelemetryData, SettingsData)
- Ошибки парсинга логируются, но не прерывают обработку
- Данные нормализованы перед передачей в callbacks
- Централизована отправка данных на устройство
- Создан приватный метод _send_to_device(topic, payload) для единой точки отправки
- Все методы отправки (send_time_to_device, send_settings_request_to_device, send_weather_to_device, send_config) теперь используют эту функцию
Результат: eliminates код дублирование, улучшает consistency и упрощает отладку
- Оптимизация логирования
- Входящие payload'ы и топики → DEBUG уровень (было INFO)
- Регистрация callbacks → DEBUG уровень
- Успешная отправка → INFO с кратким контекстом
- Подписка на топики → DEBUG уровень
2) Weather Service (adapter.py)
- Исправление фабрики адаптера
- Изменено: from_yandex(self, ...) → from_yandex(cls, ...)
- Корректно возвращаем: return cls(...) вместо return self(...)
- Bug fix: метод был объявлен как instance method, но использовался как class method
3) Redis Cache Manager (cache_manager.py)
- Удалены debug-логи
- Удален logger.info дамп полного содержимого Redis данных
- Удален traceback.print_exc() в обработчиках ошибок
Результат: чище логи в production
- Безопасное отключение от Redis
- Добавлена проверка hasattr(self.redis_client, "wait_closed") перед вызовом
- Некоторые redis-клиенты не имеют этого метода — теперь ошибка обработана gracefully
- Логируется информация об отключении
4) Worker (worker.py)
- Валидация SettingsData в методе получения конфига
- Метод get_current_config() теперь обрабатывает исключения при парсинге SettingsData(**response)
- Невалидные данные логируются с warning, возвращается None
Результат: нет crash'ей при получении поломанного payload'а от устройства
- Извлечены магические числа в константы:
DEFAULT_WEATHER_UPDATE_INTERVAL = 300 (5 минут)
DEFAULT_TIME_UPDATE_INTERVAL = 86400 (1 сутки)
DEFAULT_HEARTBEAT_INTERVAL = 60 (60 секунд)
DEFAULT_DEVICE_ID = "greenhouse_01"
Результат: легче менять параметры, понятнее код
- Улучшения типизации
- Метод handle_weather_request возвращает dict (явно указано)
- Более консистентная типизация через сигнатуры методов
5) Логирование (logger.py & Dockerfile)
- Добавлена конфигурируемая LOG_LEVEL переменная
- logger.py: Читает LOG_LEVEL из окружения (по умолчанию INFO)
- Применяет уровень ко всем обработчикам (файл + консоль)
- Поддерживает стандартные уровни: DEBUG, INFO, WARNING, ERROR, CRITICAL
- Очищен Dockerfile
- Удалено жесткое значение ENV LOG_LEVEL=DEBUG
- Теперь используется значение из .env файла
- Production-default: INFO (все DEBUG логи скрыты)
- Обновлен .env файл
- Добавлена переменная: LOG_LEVEL=INFO
- Исправлен формат (убраны пробелы: KEY=VALUE вместо KEY = VALUE)
- Понижены шумные DEBUG логи
"📡 Подписались на топик..." → DEBUG (из MQTT layer)
"✅ Установлен обработчик..." → DEBUG (из worker)
"MQTT listening task started" → DEBUG
Результат: в production (LOG_LEVEL=INFO) видны только важные события
- Удален дамп Redis данных
- Было: logger.info(f"DEBUG: Данные из Redis: {parsed}") на каждый цикл обновления
Результат: логи стали намного чище (на 80% меньше noise)

Исправление формата
🎯 Ключевые выводы
Надежность: Валидация MQTT payload'ов предотвращает crash'и от поломанных данных
Maintainability: Централизованная отправка и извлеченные константы упрощают поддержку
Observability: Чистые, структурированные логи с конфигурируемым уровнем
Production-ready: Production-default LOG_LEVEL=INFO, DEBUG доступен при необходимости

drainQueue() + sdTask else-ветка — guard cameraReady

- RECONNECT_INTERVAL 60000 → 5000 мс: при потере связи
  переподключение раз в минуту делало плату практически недоступной

  окно ожидания сокращено с 8 до 3 секунд, детектор упрощён
